# 노코드 업무 자동화 파이프라인 구현 및 도구 비교 분석

> **"코딩 없이 마우스 클릭만으로 일이 자동으로 돌아가게 만들기"**  
> 본 프로젝트는 반복적인 수작업 업무 워크플로우를 노코드 자동화 도구인 **Make**와 **Zapier**를 활용하여 조건 분기가 포함된 실제 자동화 파이프라인을 설계·구현·비교한 프로젝트입니다.

## 📋 프로젝트 개요
* **학습 목표:** 
  * Trigger(시작 이벤트)와 Action(처리 동작)의 작동 원리 이해
  * 조건 분기(Filter/Router)를 활용한 비즈니스 로직 설계
  * 자동화 도구별 특징(UI/UX, 비용, 로그 등) 비교 분석 및 적합한 도구 선정 역량 확보
* **프로젝트 기간:** 2026.06.03 ~ 06.07 (40시간)
* **제출 산출물:** 자동화 도구 비교 구현 보고서(Make vs Zapier) (프로젝트 1) & 자유 주제 자동화 설계 및 구현 보고서 (프로젝트 2)
   * 프로젝트 1: 동일한 자동화 워크플로우를 **Make**와 **Zapier** 두 가지 도구로 각각 구현하여 기능 및 핵심 지표를 비교 분석. ([보고서](project_1_tool_comparison.md) 참고)
   * 프로젝트 2: 현업에서 발생하는 실제 반복 업무를 하나 선정하여 상용 가능한 자동화 파이프라인 구축. ([보고서](project_2_implementation.md) 참고).

## 💡 결과 및 회고

본 프로젝트는 대표적인 노코드 자동화 툴인 **Make와 Zapier를 직접 경험하고 비교**하며, 실제 현업 환경에서 업무 효율성을 극대화할 수 있는 방안을 깊이 있게 고민해 보는 뜻깊은 시간이었습니다. 

두 도구 모두 강력한 기능을 제공하지만, **운영 비용과 데이터 가공의 유연성 측면에서는 Make가 더 효율적인 선택**이라는 결론을 얻었습니다.

이번 프로젝트를 발판 삼아 향후 다음과 같은 방향으로 자동화 아키텍처를 확장할 수 있을 듯 합니다.
* **하이브리드 자동화:** 노코드의 한계를 극복하기 위해 **코드 기반 자동화(Python/JavaScript)** 기술 내재화하기
* **오픈소스 툴 도입:** 비용 최적화와 자유도 높은 커스텀을 위해 Make/Zapier의 대안으로 꼽히는 **n8n** 활용해 보기
* **자체 에이전트 구축:** 클라우드 플랜 비용에 종속되지 않도록 **자체 AI 에이전트(AI Agent)**를 설계하고 업무에 직접 활용하는 역량 기르기
